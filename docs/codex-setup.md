# Codex 设置记录

本文档记录在 `mywh78-rgb/JN` 仓库中使用 Codex 的基础流程。

## 目标

通过这个仓库学习 Codex 如何读取、整理和维护项目文件，并为跨境电商项目建立基础文档结构。

## 已完成步骤

1. 授权 Codex 读取 GitHub 仓库 `mywh78-rgb/JN`。
2. 使用只读方式检查仓库内容。
3. 确认仓库默认分支为 `main`。
4. 确认初始仓库只有 `README.md` 文件。
5. 规划基础项目结构。
6. 在非 `main` 分支上创建基础文档文件。

## 推荐工作流程

1. 先让 Codex 做只读检查，确认仓库当前状态。
2. 明确本次任务目标和需要修改的文件。
3. 创建新分支，不直接修改 `main`。
4. 让 Codex 在新分支中创建或更新文件。
5. 检查变更内容。
6. 需要合并时，再创建 Pull Request。

## 注意事项

- 不要直接在 `main` 分支上做测试修改。
- 每次任务建议记录到 `tasks/` 目录。
- 重要操作前先确认计划，再执行写入。
- 如果涉及真实业务数据，应避免提交敏感信息。
