# SKU 单品项目总控交付 SOP

本文档用于串联一个商品 SKU 从前期调研、选品判断、供应链核价、商品上架、图片素材、短视频脚本、达人推广、日常运营、订单履约到商品测试复盘和店铺周报沉淀的完整交付流程。

## 一、文档用途

本 SOP 用于作为单个 SKU 项目的总控文档，帮助运营人员明确每个阶段应该做什么、输出什么资料、引用哪些文档、由谁负责、当前状态如何，以及是否具备进入下一阶段的条件。

使用目标：

- 管理单个 SKU 从 0 到测试复盘的完整链路。
- 避免选品、核价、上架、素材、达人、履约和复盘信息分散。
- 明确每个阶段的交付物和风险检查项。
- 为团队协作、周报复盘和后续放量决策提供统一记录。

## 二、适用场景

本 SOP 适用于以下场景：

- 新 SKU 启动前的项目规划
- TikTok Shop / Amazon / Shopify / Temu 商品测试
- 单品从参考图到上架测试的全流程管理
- 商品进入达人 BD、短视频测试或自然流测试前检查
- 商品测试结束后的复盘、放量、优化或暂停决策
- 团队需要对齐运营、供应链、素材、内容、达人和履约进度
